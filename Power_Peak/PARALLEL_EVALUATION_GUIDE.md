# 高速化版PER評価スクリプトの使い方

## 概要

`evaluate_per_vs_terminals_parallel.jl`は、並列実行により大幅に高速化されたPER評価スクリプトです。

## 主な改善点

### 1. 並列実行（Distributed.jl）
- CPUコア数に応じて自動的にワーカープロセスを起動（最大8ワーカー）
- `pmap`による並列シミュレーション実行
- **期待される高速化**: 4-8倍（CPUコア数に依存）

### 2. I/O最適化
- ファイル出力を完全に削除（最終結果のみ保存）
- コンソール出力を最小化
- メモリ効率の向上

### 3. 最適化されたシミュレーション
- `simulation_wrapper_fast.jl`を使用
- 同期処理の簡略化
- 不要な計算の削除

## 使用方法

### 基本実行
```bash
julia evaluate_per_vs_terminals_parallel.jl
```

### パラメータ調整

スクリプト内で以下のパラメータを変更可能：

```julia
# 端末数の範囲
TERMINAL_COUNTS = [10, 20, 30, 40, 50, 75, 100]

# 各設定での実行回数
NUM_RUNS = 30

# チャネル数の比較
CHANNEL_CONFIGS = [1, 8]
```

## 出力ファイル

### `result_evaluation/evaluation_detail_*.csv`
全実行の詳細結果：
- NumTerminals: 端末数
- NumChannels: チャネル数
- RunID: 実行番号
- TotalPackets: 総パケット数
- Success: 成功数
- Collisions: 衝突数
- PER: パケットエラー率（%）

### `result_evaluation/evaluation_summary_*.csv`
統計サマリー：
- MeanPER: 平均PER
- StdPER: 標準偏差
- CI95: 95%信頼区間
- MinPER, MaxPER: 最小・最大値
- MedianPER: 中央値
- NumRuns: 実行回数

## 性能比較

### 元のスクリプト（シーケンシャル）
- 30回実行 × 7端末設定 × 2チャネル設定 = 420回
- 1回あたり約10秒 → **合計約70分**

### 並列版（8ワーカー）
- 同じ420回の実行
- 並列実行により約8倍高速化 → **合計約9分**

## トラブルシューティング

### ワーカー数の調整
デフォルトでは最大8ワーカーですが、変更可能：

```julia
# スクリプト内の以下の行を変更
num_workers = min(Sys.CPU_THREADS, 8)  # 最大8ワーカー
# 例: num_workers = 4  # 4ワーカーに制限
```

### メモリ不足の場合
ワーカー数を減らすか、端末数を減らしてください。

## 元のスクリプトとの違い

| 機能 | 元のスクリプト | 並列版 |
|------|--------------|--------|
| 実行方式 | シーケンシャル | 並列（pmap） |
| ファイル出力 | 各実行ごと | 最終結果のみ |
| コンソール出力 | 詳細 | 最小限 |
| 実行時間（420回） | 約70分 | 約9分 |
| 結果の精度 | 同一 | 同一 |

## 注意事項

1. **初回実行**: ワーカー起動とコンパイルに時間がかかります（1-2分）
2. **メモリ使用量**: ワーカー数 × シミュレーションメモリ
3. **乱数シード**: 各実行で異なるシードを使用（再現性あり）
